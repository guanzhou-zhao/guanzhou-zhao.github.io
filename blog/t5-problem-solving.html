Got blocked on a simple problem

The problem is the framework I am using failed to work properly. I read the error message and read through the configration file and googled. Then tried all the solutions I found but the problem was still there. Then I asked a peer for help. He read through the configration file, and told me the file was OK.

According to the error message, the problem was indeed in the configration file. I felt there was nothing to do and the problem might be caused by Windows Operating System with which I couldn't do nothing.

Finally, I happened to rewrite the file again and realized that the problem was a punctuation. As I was using Chinese input and English input at the same time and Chinese punctuations are different with English punctuations, but they looked almost the same in the eclipse editor and the framework couldn't recognize Chinese ones.

I learned that there would be a solution to a problem. It might not be easy to solve it, but just keep trying.

Solved a problem in an elegant way

The problem is a bug in Global Sources's system. User could add several factories but only the first factory were showed on homepage.

I used techniques of Rubber ducky, Try something, and log. In order to resolve the problem, I read through the related codes and explained them in mind(or to Rubber ducky). After understanding the logic and function of the related code, there was a solution in mind, then I added some codes and ouput the results into log to verify my solution. Step by Step, the problem was resolved by adding several lines of code.

I felt the problem was diffcult, because there were a lot of codes to investigate. After I got the solution, I doubted that the problem wouldn't be resolved in such an easy way. But the solution worked, I felt strong sense of achievement.

I learned that some problems looked difficult, but I should not be afraid of them. I can only solve a problem by understanding it. And the resolution might be simple.

Reflect on how confident you feel using the problem solving techniques and process:

I am comportable with the techniques of Trying something, Rubber ducky, Reading error messages, Logging, Googling and Asking peers for help if it's convinient for both.

Pseudocode is a good tool and technique, especially for design, I think. But I rarely used it.

Explain to your non-tech friend about Array's methods of map(), filter(), and reduce().

Array.map() method returns a new array with same legth with original array. Array.map() method implements callback function on each item in origin array, and depending on item value and the rule in callback function produces a new item in corresponding index in new array.

Array.filter() method returns a new array with length that is shorter or equal with original array. Array.filter implements callback function on each item, and depending on item value and rule in callback function to determin if pick the item and put it into new array.

Array.reduce() function return a single object which has taken each item in original array into account.
